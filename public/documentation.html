Movie API Documentation

This API provides access to information about movies, including details such as title, genre, director, and more.

API Endpoints

Request URL Response
GET /movies Method: GET
URL: /movies
Description: Returns a list of all movies available in the database.
Query Parameters: None
Request Body Format: None
Response Data Format: JSON - An array of movie objects, each containing details such as title, genre, director, and description.
Example Response: 
[
{
    "_id": "60d0fe4f5311236168a109ca",
    "Title": "Inception",
    "Description": "A thief who steals corporate secrets through use of dream-sharing technology is given the inverse task of planting an idea into the mind of a CEO.",
    "Genre": {
    "Name": "Sci-Fi",
    "Description": "Science Fiction is a genre that uses speculative, fictional science-based depictions of phenomena that are not fully accepted by mainstream science."
    },
    "Director": {
    "Name": "Christopher Nolan",
    "Bio": "Christopher Nolan is a British-American film director, producer, and screenwriter."
    },
    "Actors": ["Leonardo DiCaprio", "Joseph Gordon-Levitt", "Elliot Page"],
    "ImagePath": "/path/to/image.jpg",
    "Featured": true
}
]
GET /movies/:title Method: GET
URL: /movies/:title
Description: Returns detailed information about a single movie by its title. This includes the movie's description, genre, director, image URL, and whether it is featured.
Query Parameters: `:title` - The title of the movie.
Request Body Format: None
Response Data Format: JSON - An object containing details about the specified movie.
Example Response: 
{
"_id": "60d0fe4f5311236168a109ca",
"Title": "Inception",
"Description": "A thief who steals corporate secrets through use of dream-sharing technology is given the inverse task of planting an idea into the mind of a CEO.",
"Genre": {
    "Name": "Sci-Fi",
    "Description": "Science Fiction is a genre that uses speculative, fictional science-based depictions of phenomena that are not fully accepted by mainstream science."
},
"Director": {
    "Name": "Christopher Nolan",
    "Bio": "Christopher Nolan is a British-American film director, producer, and screenwriter."
},
"Actors": ["Leonardo DiCaprio", "Joseph Gordon-Levitt", "Elliot Page"],
"ImagePath": "/path/to/image.jpg",
"Featured": true
}
GET /genres/:name Method: GET
URL: /genres/:name
Description: Returns data about a specific genre by its name. This includes the genre's description.
Query Parameters: `:name` - The name of the genre (e.g., "Thriller").
Request Body Format: None
Response Data Format: JSON - An object containing details about the specified genre.
Example Response: 
{
"Name": "Sci-Fi",
"Description": "Science Fiction is a genre that uses speculative, fictional science-based depictions of phenomena that are not fully accepted by mainstream science."
}
GET /directors/:name Method: GET
URL: /directors/:name
Description: Returns biographical data about a director by their name. This includes the director's biography, birth year, and death year (if applicable).
Query Parameters: `:name` - The name of the director.
Request Body Format: None
Response Data Format: JSON - An object containing biographical details about the specified director.
Example Response: 
{
"Name": "Christopher Nolan",
"Bio": "Christopher Nolan is a British-American film director, producer, and screenwriter.",
"BirthYear": "1970",
"DeathYear": null
}
POST /users Method: POST
URL: /users
Description: Allows new users to register by providing necessary user information.
Query Parameters: None
Request Body Format: JSON - User details such as username, password, email, and birthday.
Example Request Body: 
{
"Username": "johndoe",
"Password": "password123",
"Email": "johndoe@example.com",
"Birthday": "1990-01-01"
}
Response Data Format: JSON - A confirmation message indicating successful registration.
Example Response: 
{
"_id": "60d0fe4f5311236168a109cb",
"Username": "johndoe",
"Email": "johndoe@example.com",
"Birthday": "1990-01-01",
"FavoriteMovies": []
}
PUT /users/:username Method: PUT
URL: /users/:username
Description: Allows existing users to update their user information, such as their username.
Query Parameters: `:username` - The username of the user.
Request Body Format: JSON - Updated user details such as username, email, and password.
Example Request Body: 
{
"Username": "john_updated",
"Password": "newpassword123",
"Email": "newemail@example.com",
"Birthday": "1990-01-01"
}
Response Data Format: JSON - A confirmation message indicating successful update of user information.
Example Response: 
{
"_id": "60d0fe4f5311236168a109cb",
"Username": "john_updated",
"Email": "newemail@example.com",
"Birthday": "1990-01-01",
"FavoriteMovies": []
}
POST /users/:username/movies/:movieID Method: POST
URL: /users/:username/movies/:movieID
Description: Allows users to add a movie to their list of favorite movies.
Query Parameters: `:username` - The username of the user.
`:movieID` - The ID of the movie.
Request Body Format: None
Response Data Format: JSON - The updated user object with the newly added movie in their favorites.
Example Response: 
{
"_id": "60d0fe4f5311236168a109cb",
"Username": "johndoe",
"Email": "johndoe@example.com",
"Birthday": "1990-01-01",
"FavoriteMovies": [
    {
    "_id": "60d0fe4f5311236168a109ca",
    "Title": "Inception",
    "Description": "A thief who steals corporate secrets through use of dream-sharing technology is given the inverse task of planting an idea into the mind of a CEO."
    }
]
}
DELETE /users/:username/movies/:movieID Method: DELETE
URL: /users/:username/movies/:movieID
Description: Allows users to remove a movie from their list of favorite movies.
Query Parameters: `:username` - The username of the user.
`:movieID` - The ID of the movie.
Request Body Format: None
Response Data Format: JSON - The updated user object with the movie removed from their favorites.
Example Response: 
{
"_id": "60d0fe4f5311236168a109cb",
"Username": "johndoe",
"Email": "johndoe@example.com",
"Birthday": "1990-01-01",
"FavoriteMovies": []
}
DELETE /users/:username Method: DELETE
URL: /users/:username
Description: Allows existing users to deregister from the service.
Query Parameters: `:username` - The username of the user.
Request Body Format: None
Response Data Format: JSON - A confirmation message indicating successful deregistration of the user.
Example Response: 
{
"message": "User deregistered successfully"
}
